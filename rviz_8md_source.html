1 # Live Visualization with RViz {#rviz}
2 
3 Robowflex offers a helper class, robowflex::IO::RVIZHelper, which makes it easy to display information in RViz from scripts that use Robowflex.
4 This document gives a brief tutorial of how you might use RVIZHelper in your scripts.
5 You can also look at [ur5_visualization.cpp](ur5__visualization_8cpp_source.html) which demonstrates RVIZHelper.
6 
7 ## Using RVIZHelper in a Script
8 
9 You first need to create your robot and setup ROS:
10 ```cpp
11 // Startup ROS
12 ROS ros(argc, argv);
13 
14 // Create the default UR5 robot.
15 auto ur5 = std::make_shared<UR5Robot>();
16 ur5->initialize();
17 ```
18 
19 With your robot, we can create the helper:
20 ```cpp
21 IO::RVIZHelper rviz(ur5);
22 ```
23 
24 This republishes the robot's information under an easily typable name on the parameter server.
25 By default, the name is `/robowflex/robot_description` and `/robowflex/robot_description_semantic`.
26 However, you can change this if you are visualizing multiple robots:
27 ```cpp
28 IO::RVIZHelper rviz(ur5, "ur5");
29 ```
30 
31 This will publish `/ur5/robot_description` and `/ur5/robot_description_semantic`.
32 
33 You should probably insert a pause in your program at this point (i.e., `std::cin.get()`, `sleep()`, or something), as you need to setup RViz to look at these topics.
34 
35 ## Setting up RViz
36 
37 Robowflex namespaces things to avoid conflicting with other ROS happenings, but this means RViz needs a little configuration.
38 By default, the helper publishes:
39 - Planning scenes under `/<name>/scenes` with the type `moveit_msgs::PlanningScene`, displayable with the RViz `PlanningScene` widget.
40 - Trajectories under `/<name>/trajectory` with the type `moveit_msgs::DisplayTrajectory`, displayable with the RViz `Trajectory` widget.
41 - Markers under the default `/visualization_marker_array` with the type `visualization_msgs::MarkerArray`, displayable with the RViz `MarkerArray` widget.
42 
43 `PlanningScene` and `Trajectory` need the robot description topic set to whatever you are publishing the robot under (see above).
44 They also need the topic adjusted to the namespace as well.
45

51 ## What you can Display
52 
53 - Trajectories with `robowflex::IO::RVIZHelper::updateTrajectory()` and `robowflex::IO::RVIZHelper::updateTrajectories()`
54 - Planning scenes with `robowflex::IO::RVIZHelper::updateScene()` and `robowflex::IO::RVIZHelper::removeScene()`
55 - [RViz Markers](http://wiki.ros.org/rviz/DisplayTypes/Marker) (a way to show primitive shapes and other information like text or arrows in a scene), which are managed by the helper class so you can easily add and remove named markers:
56  - `robowflex::IO::RVIZHelper::addGeometryMarker()` to add any `robowflex::Geometry` as a marker.
57  - `robowflex::IO::RVIZHelper::addArrowMarker()` to add arrows.
58  - `robowflex::IO::RVIZHelper::addTextMarker()` to add front-facing text.
59  - `robowflex::IO::RVIZHelper::addGoalMarker()` to add the goal regions of a motion planning request as markers to the scene.
60  Currently only solid primitives (non-mesh) regions are displayed.
61  This also visualizes (if available) orientation constraints as arrows.
62  One larger arrow for the provided orientation, and 6 smaller arrows corresponding to the boundaries of the tolerances provided along each axis.
63  The goal is also labeled by text according to the input name.
64 
65 Note that all markers added through the `add*Marker()` functions will only be shown after `robowflex::IO::RVIZHelper::updateMarkers()` is called.
66 This function also can be called in a loop to "modify" the markers if they are updating position.
67 Markers removed with `robowflex::IO::RVIZHelper::removeMarker()` will only be removed from RViz after `updateMarkers()` is called again.
68 You can have multiple markers under one name, however all markers under that name will be removed by a `removeMarker()` call.
